## Anthropic 内部组织视角

来自 Claude Code 工程总监 Fiona Fung 在 Anthropic 官方 blog 的分享(罕见的公开组织实践披露)。

### 关键数据点 / 公司实践

1. **路线图周期**: Claude Code 团队 6 个月路线图 → 3 个月就过时 → 改为 JIT 规划
2. **代码 review 分工**: Claude 接 60-70% (style / lint / bug / 测试补全),人专攻法律合规 / 信任边界 / 品味
3. **角色边界**: 无明确职级,大量 Member of Technical Staff,PM 写代码、工程师做设计
4. **招聘策略**: "Taste is scarce, typing is not" —— 看决策能力不看产出速度
5. **管理风格**: 团队扁平 + 管理者支持各小组而非命令 + 主动删除"theater"流程

### 与同期 Boris Cherny 访谈(deepdive#4)的呼应

两人都强调:
- 别把组织看成"代码搬运工集合体"
- 工程师价值在懂用户 / 懂业务 / 定义问题
- 少招人,把 token 资源给到位
- 跨项目轮岗培养系统理解

→ Anthropic 在组织设计上有 **一致且公开的对外叙事**,这是 Anthropic 的品牌护城河之一。
